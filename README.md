# Параметрический алгоритм синусоиды

Этот документ описывает, как работает скрипт `SineWaveDrawer` в Unity.  
Система является параметрической — форма волны полностью зависит от числовых параметров.

---

# 1. Общая идея

Скрипт делает две основные вещи:

1. Рисует синусоиду с помощью `LineRenderer`
2. Двигает объект (`movingPoint`) вдоль этой синусоиды

Пересчёт выполняется автоматически в методе `Update()`.

---

# 2. Основные параметры

## Amplitude (амплитуда)

```
amplitude
```

Определяет высоту волны.

- Чем больше значение — тем выше пики
- Чем меньше — тем более плоская волна

Это вертикальный масштаб кривой.

---

## Wavelength (длина волны)

```
wavelength
```

Определяет расстояние между пиками.

- Малое значение → волна частая
- Большое значение → волна растянутая

Это горизонтальный масштаб синусоиды.

---

## Length (общая длина)

```
length
```

Определяет, насколько длинной будет вся волна.

Увеличение значения просто продлевает волну вправо.

---

## Points (количество точек)

```
points
```

Отвечает за качество отрисовки.

- Больше точек → более плавная линия
- Меньше точек → ломаная кривая

---

## Normalized Position (позиция точки)

```
normalizedPosition (0–1)
```

Определяет положение движущейся точки вдоль волны.

- 0 → начало
- 1 → конец
- 0.5 → середина

Это процент длины волны.

---

# 3. Как строится синусоида

Алгоритм построения:

1. Делим длину на равные части (по количеству points)
2. Для каждой точки вычисляем координату X
3. Высоту Y получаем через функцию синуса
4. Передаём координаты в LineRenderer

Упрощённо:

```
t = i / (points - 1)
x = t * length
y = sin(x * 2π / wavelength) * amplitude
```

Таким образом:

- amplitude влияет на высоту
- wavelength влияет на частоту
- length влияет на общую протяжённость
- points влияет на плавность

---

# 4. Движение точки по волне

Алгоритм движения:

1. Берём normalizedPosition
2. Умножаем на length
3. Вычисляем высоту через ту же формулу синуса
4. Устанавливаем позицию объекта

Упрощённо:

```
x = normalizedPosition * length
y = sin(x * 2π / wavelength) * amplitude
```

Точка всегда остаётся строго на поверхности волны.

---

# 5. Что происходит при изменении параметров

| Параметр | Результат |
|----------|-----------|
| Amplitude | Волна становится выше или ниже |
| Wavelength | Волна становится более частой или редкой |
| Length | Волна становится длиннее |
| Points | Изменяется плавность |
| Normalized Position | Точка перемещается |

---

# 6. Пошаговая логика работы

Каждый кадр выполняется:

```
1. Проверка количества точек
2. Перерисовка всей синусоиды
3. Обновление позиции движущейся точки
```

Это делает систему динамической и позволяет менять параметры прямо во время работы.

---

# 7. Итог

`SineWaveDrawer` — это пример процедурной геометрии.

Форма кривой:

- задаётся математической функцией
- управляется параметрами
- автоматически пересчитывается

Это простая, но наглядная реализация параметрического алгоритма в Unity.

# Параметрическая модель «Стол» и «Стул»

В этом документе описано, как работают параметрические модели стола и стула в Unity.  
Идея простая: мы меняем несколько основных параметров — и объект автоматически перестраивается.

---

# 1. Что такое параметрическая модель?

Параметрическая модель — это объект, который меняется в зависимости от числовых значений.

Например:

- меняем высоту → ножки автоматически удлиняются
- меняем ширину → ножки раздвигаются
- меняем угол → спинка наклоняется

---

# 2. Стол

## Основные параметры

Самые важные параметры:

- `height` — высота стола (обычно 740–760 мм)
- `width` — ширина столешницы
- `depth` — глубина столешницы
- `tabletopThickness` — толщина столешницы

Когда мы меняем высоту:

- столешница поднимается выше
- ножки автоматически становятся длиннее

Если включён `ergonomicMode`, высота автоматически ограничивается диапазоном 740–760 мм.

---

## Как рассчитываются ножки

Высота ножек:

```
legHeight = height - tabletopThickness
```

То есть ножка — это всё расстояние до столешницы, минус её толщина.

Ножки автоматически располагаются по углам стола и сдвигаются внутрь на небольшой отступ.

---

## Что происходит при изменении параметров

Если изменить:

- **height** → ножки станут длиннее или короче
- **width** → ножки разъедутся шире
- **depth** → ножки раздвинутся по глубине

Все изменения происходят автоматически.

# 3. Стул

## Основные параметры

Самые важные параметры:

- `seatHeight` — высота сиденья (400–450 мм)
- `seatWidth` — ширина сиденья
- `seatDepth` — глубина сиденья
- `backrestHeight` — высота спинки
- `backrestAngle` — угол наклона спинки

---

## Как работает высота стула

Высота ножек считается так:

```
legHeight = seatHeight - seatThickness
```

Когда меняется `seatHeight`:

- сиденье поднимается
- ножки автоматически становятся длиннее

Высота ограничена диапазоном 400–450 мм.

---

## Как работает спинка

Спинка:

- размещается сзади сиденья
- может наклоняться (угол 90–110°)
- автоматически масштабируется по ширине сиденья

Если изменить угол — спинка наклонится назад или вперёд.

---

## Особенность задних ножек

У стула можно включить удлинение задних ножек (`autoLegLength`).  
Тогда задние ножки становятся длиннее и поддерживают спинку.

# 4. Связь стола и стула

Чтобы за столом было удобно сидеть:

```
Разница = высота стола − высота сиденья
```

Комфортный диапазон — примерно 270–320 мм.

Если разница слишком маленькая — будет тесно.  
Если слишком большая — сидеть неудобно.

---

# 5. Главное

Стол:
- главный параметр — высота
- ножки всегда подстраиваются автоматически

Стул:
- главный параметр — высота сиденья
- спинка наклоняется углом
- задние ножки могут удлиняться

Всё работает автоматически — достаточно менять параметры.

---

# Итог

Это простая параметрическая система:

- меняем параметры
- Unity пересчитывает размеры
- модель перестраивается без ручного редактирования

Такой подход удобен для прототипирования мебели и экспериментов с размерами.
